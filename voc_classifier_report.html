PM VoC 자동 분류기 - 제출 리포트

1. 프로젝트 개요

다채널(슬랙/CS티켓/앱스토어/인앱설문)로 수집된 VoC 300건을 규칙 기반으로 자동 분류하고, 즉시 대응이 필요한 이슈와 제품 개선 방향을 자동 산출하는 파이프라인입니다. classified_voc.csv(분류 결과) → voc_report.md(요약 리포트) → app.py(Streamlit 대시보드) 순으로 구성됩니다.

2. 동작 화면 URL

https://voc-classifier.streamlit.app/

4. 분류 기준

날짜 정제 기준

  • `YYYY-MM-DD`, `YYYY/MM/DD`, `M월 D일` 등 혼재된 원본 날짜 포맷을 확인함.
  • 최종 산출물(classified_voc.csv, voc_report.md)에는 날짜 컬럼을 포함하지 않으므로 별도 정제·표준화는 적용하지 않음.

결측값 처리 기준

  • `channel`이 비어 있으면 "미확인"으로 대체.
  • `user_type`이 비어 있으면 "미확인"으로 대체(즉시 대응 우선순위 가점 대상에서는 제외).
  • `content`, `severity_hint`는 결측 없이 원본 값 그대로 사용.

중복 데이터 처리 기준

  • id가 다르면 내용이 동일해도 별건으로 간주하여 제거하지 않음(예: F012, F300).
  • 원본 300행 = classified_voc.csv 300행을 그대로 유지, 중복 제거로 인한 행 수 변경 없음.

유형 분류 기준

  • 키워드 기반 규칙으로 1차 세부 분류(버그신고/결제·과금 오류/결제·과금/기능요청/칭찬/사용문의/기타) 후, 과제 기준 4종으로 매핑.
  • 버그신고, 결제·과금 오류 → 버그
  • 사용문의, 결제·과금 → 일반문의
  • 기능요청 → 기능요청
  • 칭찬 → 칭찬
  • 판단 우선순위: 결제 키워드+오류 신호 → 버그 키워드 → 기능요청 키워드 → 칭찬 키워드 → 문의 키워드 순으로 첫 매칭 규칙 적용.

감정 분류 기준

  • 유형이 버그(버그신고/결제·과금 오류)인 경우 감정은 항상 부정.
  • 해지·환불·분쟁 등 강한 이관(escalation) 키워드가 있으면 부정.
  • 그 외에는 긍정 키워드와 부정(버그) 키워드 존재 여부를 비교해 긍정/부정 판단.
  • 위 조건에 해당하지 않으면 중립(문의성/사실 전달성 문장 등).

경계 케이스 처리 기준

  • 결제 키워드와 버그 키워드가 동시에 나타나면 "결제·과금 오류"로 우선 분류 후 최종 유형은 버그로 매핑.
  • 키워드 규칙에 전혀 매칭되지 않는 애매한 건("기타")은 일반문의로 귀속.
  • 감정과 유형이 상충하는 경우(예: 긍정 키워드 + 버그 키워드 동시 등장)는 버그/이관 키워드를 우선하여 부정으로 판정.

5. 즉시 대응 TOP3 선별 기준

  • 점수 = severity_hint(높음 +3, 보통 +1) + 감정 부정 +2 + user_type이 유료/기업고객 +1 + 긴급 키워드(환불·결제·구독·오류·장애·로그인·데이터·유실·삭제·저장·동기화·복구·불편 등) 매칭 시 +1.
  • 동점 처리 순서: severity_hint 높음 우선 → 감정 부정 우선 → 매칭 긴급 키워드 수 많은 순 → id 순.
  • 상위 3건만 선정하며, 각 건에 채널/사용자 구분/심각도/유형/감정/원문/선별 이유를 함께 기록.
